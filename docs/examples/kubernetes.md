# Kubernetes CronJobs

Use these examples before deploying or reviewing Kubernetes CronJob manifests.

## Preview a timezone-aware Kubernetes CronJob

Before deploying a CronJob with `.spec.timeZone`, preview the next fire times.

```sh
croncheck next "0 9 * * *" \
  --tz America/New_York \
  --from 2024-03-08 \
  --count 5
```

This is useful around DST changes because the UTC offset can change while the
local wall time stays at 9:00 AM.

## Analyze Kubernetes CronJobs before deployment

Run checks against a manifest generated by Kustomize or Helm.

```sh
kustomize build ./deploy/prod > /tmp/cronjobs.yaml
croncheck check --from-k8s /tmp/cronjobs.yaml --window 30d
```

For machine-readable CI output:

```sh
croncheck check --from-k8s /tmp/cronjobs.yaml --window 30d --format json
```
